阶段 2：数据库建模与 JWT 鉴权 (Auth & DB)
文件名：prompt_02_auth.md

提示词内容： 基于第一阶段的项目，请实现用户系统和鉴权逻辑。

要求：

SQL 迁移： 创建 users 表。字段：id (Binary(16) 存储 UUID), username (唯一), password_hash。

认证模块：

使用 bcrypt 处理密码加密。

实现 /api/auth/register 和 /api/auth/login。

登录成功后返回有效期 24 小时的 JWT Token。

Axum 中间件： 实现一个 AuthMiddleware。它拦截请求头中的 Authorization: Bearer <token>，验证通过后将 user_id 注入到请求上下文中（Request Extension）。

请输出： src/models/user.rs、src/handlers/auth.rs 以及中间件的 Rust 实现代码。

阶段 3：文章逻辑与“无前缀”路由 (Core Logic)
文件名：prompt_03_articles.md

提示词内容： 现在实现文章的核心功能，特别注意路由设计。

需求：

数据库表： articles 表。字段：id (INT AI), author_id (关联用户), slug (唯一字符串), title, content (TEXT), is_public (Boolean)。

私有管理接口 (需 Auth 中间件)：

POST /api/articles: 创建文章。

PUT /api/articles/:id: 更新内容或切换 is_public 状态（需检查当前用户是否为作者）。

DELETE /api/articles/:id: 删除文章。

核心公开接口：

GET /:slug: 这是关键路由。根据路径中的 slug 查询文章。

逻辑检查： > * 如果文章不存在，返回 404。

如果文章 is_public == false 且请求者未登录或不是作者，返回 404/403。

如果文章公开，直接返回纯文本/JSON。

请输出： 文章相关的 Handler 代码以及如何在 main.rs 中配置这种“无前缀”路由。
